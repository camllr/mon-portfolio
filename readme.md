# Portfolio – Camille Rostan

Ce dépôt contient mon premier portfolio web, réalisé pour présenter mon profil et mes projets dans le cadre de ma recherche d'alternance en développement informatique.
Il est également possible de retrouver mes 3 anciennes versions de portfolio sur ce github : `https://github.com/camllr/mon-portfolio-evolutions`

## Contexte et objectif

- Je suis actuellement en BTS MCO, une formation qui n'est pas orientée informatique.
- En parallèle, je me forme en autodidacte au développement web afin de préparer une future entrée en BTS SIO option SLAM et d'être légitime pour une alternance.
- Ce portfolio a pour but de montrer ma motivation, ma capacité d'apprentissage et mes premières compétences techniques.

## Technologies utilisées

- **HTML5** pour la structure des pages
- **CSS3** pour la mise en forme, le design responsive et la gestion de la typographie (polices personnalisées, hiérarchie visuelle)
- **JavaScript** pour les interactions et le dynamisme du site

Il s’agit de mon tout premier projet avec ces technologies.

## Accès au portfolio

- Version en ligne : `https://camllr.github.io/mon-portfolio/`
- Code source : ce dépôt GitHub

## Contenu du portfolio

Le portfolio est organisé en plusieurs sections :

- **Hero section** : mise en avant de mon nom, de mon rôle et d’un appel à l’action pour découvrir mon profil.
- **Skills** : présentation de mes compétences principales (HTML, CSS, JavaScript) avec un focus sur le développement web.
- **Projets** : sélection de mes premiers projets, avec une description courte et des liens vers les dépôts GitHub correspondants.
- **Storytelling / Parcours** : récapitulatif de mon parcours scolaire (BTS MCO) et de mes expériences professionnelles, ainsi que ma transition vers l’informatique.
- **Contact** : liens vers mes moyens de contact (e-mail, réseaux professionnels, etc.) pour faciliter les échanges dans le cadre d’une alternance.

## À propos du développement

- Ce projet a été réalisé sans cours d'informatique, uniquement à partir de ressources en ligne et de pratique personnelle.
- Les premiers essais ont été faits hors Git, c’est pourquoi ce dépôt ne contient pas d’historique de commits détaillé.
- Pour tous mes projets suivants, j’utilise désormais Git et GitHub dès le début afin de suivre l’évolution de mon code et de mon niveau.

## Évolutions prévues

- Améliorer progressivement le design, l’accessibilité et l’ergonomie.
- Ajouter régulièrement de nouveaux projets issus de ma formation autodidacte.
- Intégrer de nouvelles fonctionnalités (animations, formulaires avancés, optimisation mobile, etc.).
